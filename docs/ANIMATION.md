# План по анимациям

Важная мысль: **большинство «анимаций» ловушек — это НЕ анимация меша, а
код/физика в Godot.** Не нужно риггить утюг, чтобы он падал — это RigidBody +
tween. Делим на 4 типа по способу производства, от самого дешёвого к дорогому.

---

## Тип 1. Физика (RigidBody) — падения, качения, отлёты

**Где делать:** прямо в Godot, модель НЕ трогаем.
**Как:** предмет = `RigidBody3D`, спит (`sleeping=true`) до триггера, потом
`apply_impulse`/`freeze=false`. Гравитация делает всё сама.

Покрывает: падающая люстра/шкаф/утюг/гиря, шар для боулинга, банан-отлёт,
шарики, катящиеся предметы, обрушение полки, маятник (PinJoint3D).
**Это ~60% всех «анимаций» ловушек. Ноль работы художника.**

## Тип 2. Твины (код) — открывания, срабатывания, тряска

**Где делать:** в Godot, `create_tween()`.
**Как:** программная анимация трансформа/материала за пару строк.

Покрывает: ящик выдвигается, дверь падает с петель, мышеловка защёлкивается
(поворот дужки), пружина дивана-катапульты, тряска мебели при обыске (уже есть),
качание люстры перед падением (уже есть), пульсация приманки.

## Тип 3. Партиклы + шейдеры — вода, огонь, ток, дым, молоко

**Где делать:** в Godot, `GPUParticles3D` + `ParticleProcessMaterial`.
**Как:** уже частично есть (`_burst`, `_cloud`, огонь). Расширяем.

Покрывает: разлив воды/молока (decal + партиклы), пламя, электродуга
(партиклы + мигающий OmniLight), дым/мука/краска-облако, брызги, искры,
пар. **Тоже без художника — чистый Godot.**

Апгрейд качества: лужи — как **Decal** на полу с нормалью + отражением;
электродуга — линия-меш с шумовым шейдером; огонь — sprite-sheet flipbook.

## Тип 4. Скелетная анимация — ТОЛЬКО персонажи

**Это единственное, что требует рига и настоящих анимаций.**
Предметы сюда почти не попадают (разве что сложная механическая ловушка).

### Персонажи — пайплайн

Наши 3D-модели-предметы генерятся статикой, а персонажам нужен скелет +
клипы (idle / бег / присед / ставит ловушку / взлом / связан / падение).

**Вариант А (рекомендую для старта) — готовый риггованный + Mixamo:**
1. Берём CC0-персонажа, совместимого с Mixamo-скелетом (или любой humanoid).
2. Заливаем на [mixamo.com](https://www.mixamo.com) → авто-риг → качаем клипы
   (Idle, Run, Crouch, Sitting, Falling, «Picking Up», «Pushing») бесплатно.
3. В Godot: один `AnimationTree` со `StateMachine` + бленд-пространство
   бег/ходьба, оверлеи-действия поверх.

**Вариант Б — AI-генерация персонажа с авто-ригом:**
- **Meshy** (Character mode) и **Nilo** умеют авто-риг + базовые анимации.
- **Anything World**, **Tripo** — авто-риг статичной модели.
- Качество анимаций пока хуже Mixamo — годится для прототипа/скинов.

**Вариант В — своя анимация в Blender:**
- Только если нужен уникальный кадр (прыжок в мешок, победная поза).
- Долго; оставляем на потом, точечно.

### Инструменты озвучены — что выбрать нам

| Задача | Инструмент | Стоимость |
|---|---|---|
| Падения/качения/отлёты | Godot RigidBody | 0 (код) |
| Открывания/защёлки/тряска | Godot Tween | 0 (код) |
| Вода/огонь/ток/дым | Godot Particles+Shader | 0 (код) |
| Анимации персонажей | **Mixamo** | 0 (бесплатно для игр) |
| Авто-риг AI-персонажа | Meshy/Nilo/Tripo | фри-кредиты |
| Уникальные ключевые кадры | Blender | 0 (время) |

## Связки: как «два предмета выглядят вместе» (верёвка/провод)

**Решение: соединения — процедурные (код), НЕ отдельные модели.** Верёвка
между растяжкой и ведром/люстрой рисуется `PropFX.build_rope(a, b, sag)`:
провисает дугой (катенария) + узлы на концах. Точки крепления берём из
габаритов модели (верх ведра = ручка, верх шкафа и т.д.).

Почему не генерить отдельные промпты под связки: соединение **динамическое** —
дистанция и угол каждый раз разные (игрок сам тянет). Готовая модель
«верёвка+ведро» подошла бы лишь для одной дистанции. Один код-helper = все
комбинации бесплатно.

Опционально позже: «подвешенные» варианты моделей (ведро уже с дужкой, люстра
с цепью) — только косметика, базовые модели уже работают.

## Что делать прямо сейчас (порядок)

1. **Перевести существующие ловушки на RigidBody** там, где сейчас tween-хак
   (люстра, шкаф уже частично) — падения станут «настоящими» и разными.
2. **Улучшить партиклы** воды/молока/огня/тока до Decal+flipbook — это даёт
   самый заметный визуальный апгрейд без единой модели.
3. **Мышеловка-защёлка, ящик-сюрприз, дверь-с-петель** — твины на 10 строк.
4. **Персонажи** — отдельный заход: берём Mixamo, ставим AnimationTree.
   Пока играбельно и с процедурными.

**Вывод:** 80% «кучи анимаций», которые ты хочешь, делаются в самом Godot
кодом и физикой — без художника и без ожидания генерации. Реальная внешняя
зависимость только одна: **анимации персонажей → Mixamo (бесплатно)**.
